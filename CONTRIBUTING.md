# Contributing

Thank you for your interest in contributing. This file provides a minimal workflow to get changes accepted quickly.

1. Read governance docs: `docs/naming-conventions.md` and `docs/branch-policy.md`.
2. Open an issue describing the change or bug before a large PR.
3. Create a branch named `fix/<short-desc>` or `feat/<short-desc>` from `main`.
4. Run any linters or tests available in the target subproject.
5. Submit a PR and link the issue; use small, focused PRs.

Formatting and checks:
- Use `snake_case` for file names and `kebab-case` for CLI scripts when applicable.
- Put model/feature versions in configs (do not hardcode in filenames).

For detailed contribution rules, refer to `docs/`.
